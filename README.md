# auto-translation

Scrape Traditional-Chinese web novels from `xbanxia.cc` and translate them to
publication-quality English.

## How it works

The pipeline lives entirely in two project skills under `.claude/skills/` — there
are no loose scripts at the repo root:

- **`scrape-translate/`** — end-to-end: book URL → scrape every chapter → translate.
  - `scripts/launch_chrome.py` — opens a real Chrome on `--remote-debugging-port=9222`
    (profile in `cf_profile/`, gitignored) and clears Cloudflare.
  - `scripts/scrape_book.py` — connects over CDP and writes `<Book Title>/chapters/*.txt`,
    `FULL_BOOK.txt`, `manifest.json`.
  - `scripts/assemble_en.py` — stitches the English chapters into `FULL_BOOK_EN.txt`
    plus a table of contents, and prints a QA report.
- **`cn-en-translator/`** — translation rules + the living name/term glossary tuned for the
  《重回真千金認親前[年代]》 novel (kept for reference / future re-scrapes).

See each skill's `SKILL.md` for the full workflow.

## Layout

```
<Book Title>/        one self-contained folder per scraped book
                     (chapters/, english/, FULL_BOOK*.txt, GLOSSARY.md, manifest.json)
reader/              Next.js reader app for browsing the translations
.claude/skills/      the scraper + translator pipeline
```

`cf_profile/` (the scraper's Chrome profile) is kept locally but gitignored — it's
regenerated by `launch_chrome.py`.
